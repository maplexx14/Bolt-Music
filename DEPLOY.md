# Развертывание (Деплой) сервиса

Этот документ описывает процесс развертывания Music Service на хостинг-сервере.

## Шаг 1: Подготовка сервера
Убедитесь, что на сервере установлены `docker` и `docker-compose`. 

## Шаг 2: Получение кода
Клонируйте репозиторий на сервер или скопируйте файлы любым доступным способом.

```bash
git clone <ваш_репозиторий>
cd music-service
```

## Шаг 3: Настройка окружения
Скопируйте пример файла конфигурации:

```bash
cp .env.example .env
```

Отредактируйте `.env` файл, обязательно изменив:
- `DB_PASSWORD` - сложный пароль для базы данных PostgreSQL
- `SECRET_KEY` - ключ для подписи токенов (можно сгенерировать командой `openssl rand -hex 32` или `python -c "import secrets; print(secrets.token_hex(32))"`)
- Вы можете также изменить значение `VITE_API_URL`, если ваш API находится по другому базовому пути, но по умолчанию прокси в Nginx сам распределит маршруты.

> **Опционально — CDN для медиафайлов:**  
> Если хотите раздавать аудио и обложки через CDN, раскомментируйте и задайте `VITE_CDN_URL` в `.env`:
> ```
> VITE_CDN_URL=https://cdn.yourdomain.com
> ```
> Фронтенд автоматически будет формировать URL вида `https://cdn.yourdomain.com/music_files/<uuid>.mp3`.  
> Убедитесь, что CDN проксирует/зеркалирует пути `/music_files/` и `/cover_files/` с вашего сервера.  
> Если переменная не задана — файлы раздаются напрямую через Nginx (поведение по умолчанию).

## Шаг 4: Запуск
Соберите образы и запустите контейнеры в фоновом режиме:

```bash
docker-compose -f docker-compose.prod.yml build
docker-compose -f docker-compose.prod.yml up -d
```

### Архитектура:
- **frontend/Nginx**: Фронтенд собирается на этапе билда (React Vite), и статические файлы теперь напрямую раздаются через отдельный Nginx (в контейнере `nginx` на порту 80/443). Это же Nginx приложение перенаправляет запросы в `/api/` непосредственно к бэкенду. 
- **backend**: Бэкенд контейнер теперь использует Gunicorn c Uvicorn worker-ами (4 процесса вместо 1), что позволяет обслуживать несколько запросов одновременно и обеспечивает Production-ready стабильность.
- **Статические файлы (Аудио и Обложки)**: Nginx в контейнере `nginx` напрямую читает тома `music_files_prod` и `cover_files_prod`. Специальные аудо-заголовки прописаны в его конифге для поддержки Range Requests, перемотки в браузере, и снижения нагрузки на FastAPI.

## Управление
Остановить и удалить контейнеры:
```bash
docker-compose -f docker-compose.prod.yml down
```

Смотреть логи:
```bash
docker-compose -f docker-compose.prod.yml logs -f
```
